# USL File Index

Index of key files/folders so any AI platform knows what exists, where the source
of truth is, and what is safe to hand to an AI tool. Update when artifacts are
added or sensitivity changes.

**Sensitivity** key: Public, Internal, Sensitive, Controlled (CUI/ITAR/TDP/etc.).
Controlled items are never safe for AI upload.

| File or folder | System | Purpose | Sensitivity | Source of truth? | Safe for AI upload? | Notes |
| --- | --- | --- | --- | --- | --- | --- |
| `context/USL_CONTEXT_SNAPSHOT.md` | Repo | Current posture/status (generated) | Internal | No (generated) | Yes | Regenerated by build script |
| `context/USL_MEMORY_LEDGER.md` | Repo | Durable operating memory | Internal | Yes (memory) | Yes | Decisions, gates, restrictions |
| `context/USL_ACTIVE_TASKS.md` | Repo | In-flight tasks | Internal | Mirrors CRM | Yes | CRM is structured record |
| `context/USL_FILE_INDEX.md` | Repo | This index | Internal | Yes (index) | Yes | — |
| `context/handoffs/` | Repo | Session handoffs | Internal | Yes (handoffs) | Yes | One file per session |
| `AGENTS.md` / `CLAUDE.md` | Repo | Shared + Claude agent instructions | Internal | Yes (instructions) | Yes | — |
| `.chatgpt/` `.codex/` | Repo | Platform instructions | Internal | Yes (instructions) | Yes | — |
| `docs/` | Repo | Operating docs / SOPs | Internal | Yes (docs) | Yes | Bus + tool docs |
| `scripts/`, `tools/` | Repo | Context-bus tooling | Internal | Yes (tooling) | Yes | Stdlib/PowerShell only |
| Local CRM database | Local CRM | Structured project state | Sensitive | Yes (structured) | No | Do not mutate from the bus |
| Local CAD repo | Local CAD | CAD / drawings | Controlled | Execution system | No | Controlled; never upload |
| Google Drive `USL Command Drive/` | Drive | Artifact/large-file store | Mixed | Yes (artifacts) | Cleared artifacts only | Not a memory layer |
| Supplier records / pricing | CRM/Drive | Supplier data | Sensitive/Controlled | CRM | No | Restricted |

## Rules

- Anything marked **Controlled** or supplier-restricted is never uploaded to an AI
  tool and never committed to the repo.
- Google Drive holds only artifacts cleared for storage — it is not memory.
